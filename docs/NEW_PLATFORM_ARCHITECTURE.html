3 · How you switch (bottom-left, always there)
↗ Topbar (logo · search · bell · user)
⇄ Switch to New Platform
Currently in: Command Center

Bottom-left sidebar block

Users with access to both platforms see a switcher pinned to the bottom-left of the sidebar — visible from every page. One click toggles between Command Center ↔ New Platform without re-authenticating. The cookie that remembers the picker choice is updated, so a fresh tab opens to the last-used platform.

Users with access to only one platform never see the switcher (and don't see the picker on sign-in either). Clean, no noise.

5 · Strict separation (so the two platforms can't entangle)

🔒 Hard rules — enforced everywhere

Backend code: Command Center under lib/salesforce/* + existing libs. New Platform under lib/commercial/*. Zero cross-imports.
Database: Command Center keeps its existing tables. New Platform tables all prefix commercial_*. No shared FKs (except profiles.user_id for identity).
Routes: Command Center at /dashboard/*; New Platform at /dashboard/commercial/*. Middleware enforces access.
Sidebar & nav: Two separate sidebar components. Each platform's nav can't surface the other's pages.
Notifications: Reused bell, but notification kind values are platform-prefixed (e.g. cc.customer_form_submitted vs np.bid_received).
Invoices: Command Center hours → PPP invoice. New Platform hours → Alex invoice. No mixing.
Memory / docs: Separate memory files + separate plan docs. PPP doc names start project_ppp_*; New Platform start project_np_*.
Audit trail: Command Center uses existing audit pattern. New Platform has its own commercial_audit_log table covering every commercial_* write.
6 · What IS shared (the thin hub layer)

🔗 Reused across both — the only places they touch

AuthSupabase Google SSO + profiles table. Two new flags: has_command_center_access, has_new_platform_access.
ShellTopbar + sidebar layout primitives. Each platform supplies its own sidebar items.
Notification bellOne bell component. Each platform pushes its own kinds; both render in the same dropdown when user has both.
Mobile tokensiOS HIG 44px taps, safe-area-inset, font scale, blue+emerald palette.
Document storageSupabase Storage. Separate buckets (ppp-documents, commercial-documents) but same upload/signed-URL pattern.
PDF + emailSame Resend account + PDF tooling. Templates per-platform.
SearchTwo separate global-search indexes. Picker shows only what the viewer can access.
Customer-form patternReused for Command Center color forms AND New Platform public bid submission. Token-gated, mobile-first.
7 · Build order (next 5–6 weeks)

📋 Phase by phase — what ships when

  1. Now (Command Center): Materials ordering page speed pass — completes the residential side polish. (bills to PPP invoice)
  2. Phase 0 — Foundation (1–2 days): Platform picker, RBAC fork, bottom-left switcher, shared shell wiring, commercial_* table conventions, commercial_audit_log. Blocks everything else.
  3. Phase 1 — Account Management (2–3 days): Commercial accounts + contacts + documents + perf dashboard.
  4. Phase 2 — Opportunity (3–4 days): Bid pipeline, plans/specs, inbound email parser, online bid submission form.
  5. Phase 3 — Estimating & Proposal (3–4 days): Estimate workbench, proposal versions, RFIs, bid outcome.
  6. Phase 4 — Contract Award (1–2 days): Contract entry, submittals planning, procurement, requirements.
  7. Phase 5 — Project Setup (2 days): Project record, team picker (multi-PM), documents, submittals approval flow.
  8. Phase 8 — Billing & Financials (3–4 days) ⬆️ PULLED UP: Multi-invoice with start/end + paid filter (Karan's explicit ask). Lets PPP invoice commercial work the moment a contract is signed.
  9. Phase 6 — Project Execution (4–5 days): Daily reports (mobile + offline), labor/material/equipment/sub, quality, punchlist, schedule. Heaviest.
  10. Phase 7 — Change Management (2 days): CO pipeline, impact tracking, contract revision rollup.
  11. Phase 9 — Closeout (2 days): Punchlists, warranty docs, lessons learned.
  12. Executive Dashboards (3 days): Sales / Operations / Financial / Workforce / Company Scorecard.

Total: ~26–34 build-days for full scope. MVP cutline (Phases 0+1+2+8) ships in ~2 weeks if we need to compress.
