One system to run every commercial job, end to end.

The pitch in one paragraph

Today PPP's commercial side lives in email threads, spreadsheets, and PDFs. The Commercial Command Center pulls all of it into one place — from the moment a lead comes in to the day the final invoice is paid. Same login as the existing PPP Command Center; both platforms behind one door. Built specifically for PPP — every screen designed around how Alex's team actually works.

What it covers — the full job lifecycle

Nine connected phases. Each phase is its own build slice we ship, test, and turn on without breaking the others.

What Alex gets, phase by phase

PHASE 1

Account Management

Shipping now

The customer directory + every detail PPP needs to do business with them.

Alex gets
  • One record per commercial company — billing address, AP contact, industry, rating
  • Contacts list with roles (Decision Maker, PM, Billing, Site, etc.)
  • Which PPP staff are assigned, with primary holder flags
  • Documents on file — COI, W-9, MSA, prequal — with expiry alerts before they lapse
  • One-glance Account 360 showing every count + last activity
PHASE 2

Opportunity (Preconstruction)

NextMVP path

The deal pipeline. From first email to "we won it."

Alex gets
  • Kanban pipeline: Lead → Qualified → Walkthrough → Estimating → Proposed → Won/Lost
  • Forward an RFP email to a special address → auto-creates the opportunity
  • Walkthrough notes + geo-tagged photos from the field
  • RFI tracker (questions to GC/architect during bid)
  • Win/loss reasons + "lost to" competitor field for analysis
  • Auto-reminders when a bid is pending too long
  • Sales pipeline $$ value rolls up to dashboards + Account 360
PHASE 3

Estimating & Proposal

Queued

Build a bid faster, with less guesswork, and never lose money on it.

Alex gets
  • Line-item bid workbench: area × sqft × finish × paint product × labor × markup
  • Subcontractor sub-bids rolled in (drywall, electrical, sub trades)
  • Multi-revision comparison (v1 vs v2 side-by-side)
  • Estimate library — clone from any past project
  • Cost-per-sqft analytics ("this bid is +12% vs your last 5 hospital projects")
  • One-click proposal PDF builder with cover letter, schedule of values, exclusions
PHASE 4

Contract Award

Queued

Sign and lock — no more chasing PDFs.

Alex gets
  • Contract upload + Notice to Proceed
  • Insurance binders + bonds attached + verified expiry
  • Schedule of values locked (no edits without a Change Order)
  • Sign-off workflow: PM → ops → Alex with full audit trail
PHASE 5

Project Setup

Queued

Spin up the project. Everyone knows who's on it.

Alex gets
  • Project record built straight from the won opportunity
  • Multi-PM team per project (primary + assistant PMs, super, foreman)
  • Crew assignments + skill tags
  • Budget freeze locks the won estimate as the baseline
  • Subcontractor onboarding (their COIs, payment terms)
  • Permits, OSHA, safety docs tracker
  • Mobilization + kickoff checklist
PHASE 6

Project Execution

Queued · heaviest

The day-to-day. Mobile-first because foremen live on iPhones.

Alex gets
  • Daily reports submitted from the foreman's phone — geo-tagged photos, hours, weather, delays
  • Labor + material consumption logged per area
  • Quality checks + per-area sign-off
  • Punchlist with photos, assignee, due date
  • Schedule view (Gantt or weekly grid)
  • Owner/GC daily report auto-distributed
  • Customer portal (read-only) so the owner can see progress photos
  • Real-time budget vs actual per project — the "are we making money?" answer
PHASE 7

Change Management

Queued

Catch every change order. Get paid for every one of them.

Alex gets
  • CO pipeline: Request → Estimate → Customer Approval → Contract Amendment → Execute
  • Impact tracking: budget delta, schedule delta
  • CO log per project with running total
  • Margin on COs reported separately (typically higher than base contract)
PHASE 8

Billing & Financials

QueuedMVP path

Send invoices, take payments, see your cash position — without leaving the platform.

Alex gets
  • Multi-invoice per project (progress billing — standard for commercial)
  • AIA G702/G703 invoice templates with schedule of values + % complete per line
  • Retainage hold + release (5% default, configurable)
  • Payment recording (full or partial)
  • AR aging by account and by project
  • Auto past-due reminders at 15 days
  • Lien waivers collected on each payment
  • Project P&L: estimate vs actual variance report
  • Numbers flow back to the Account 360 — Alex sees customer balances at a glance
PHASE 9

Closeout

Queued

Wrap the job clean. Set up the next one.

Alex gets
  • Punchlist completion sign-off
  • Warranty docs + as-built drawings upload
  • Final lien releases collected
  • Customer satisfaction survey
  • Project archive (ZIP of every doc, daily report, photo)
  • Final P&L + estimate-vs-actual variance
  • Lessons learned doc → flows back into the estimating library

How it fits with the existing PPP Command Center

PPP staff sign in once. The platform picker asks which side they're using — PPP Command Center (the existing residential / Salesforce side) or Commercial Command Center (this build). Same audit, same notification bell, same brand. Different data, different workflows. Both fully separated in the backend so changes to one can't break the other.
